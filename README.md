# index-web-journal

## 项目简介

这是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为一系列静态页面内容的集中存放与版本管理空间。你可以将其理解为一个网页日志或页面集合的存档点。

## 目录结构说明

```
.
├── pages/          # 存放独立的 HTML 页面文件
├── assets/         # 页面引用的静态资源（如 CSS、JS、图片等）
├── archive/        # 历史版本或已归档的页面文件
└── README.md       # 本文件
```

各目录用途如下：

- **pages/**：当前活跃或最新版本的 HTML 页面。每个页面通常为一个独立文件或一个子目录。
- **assets/**：页面所依赖的通用资源文件，可按类型或页面组织子目录。
- **archive/**：不再更新或已替换的旧版本页面，用于保留历史记录。

## 页面归档说明

本仓库中的页面内容可能包含个人笔记、实验性设计、临时展示或学习记录等。每个页面均独立存在，互不依赖。页面命名建议遵循以下规则：

- 使用有意义的英文或拼音命名，避免特殊字符。
- 可在文件名中加入日期或版本号，便于区分迭代。

例如：`note-2025-01.html`、`experiment-v2.html`。
